BÀI 4 – LỚP VÀ ĐỐI TƯỢNG

Mục tiêu học tập

  • Hiểu rõ khái niệm lớp (class) và đối tượng (object) trong Java.
  • Nắm cấu trúc, phạm vi truy cập và cách khai báo lớp, phương thức, thuộc tính.
  • Thành thạo cách sử dụng constructor, destructor, từ khóa this, static, và overloading.
  • Phân tích các vấn đề thiết kế hướng đối tượng: encapsulation, trách nhiệm lớp, reuse code.

1. Khái niệm lớp và đối tượng

Một lớp (class) là khuôn mẫu (template) để tạo ra các đối tượng. Một đối tượng (object) là thể hiện cụ thể của lớp đó.

class Student { String name; int age; void show(){ System.out.println(name + " - " + age); } }
Student s1 = new Student(); s1.name = "Minh"; s1.age = 20; s1.show();

2. Thành phần của lớp

  • Thuộc tính (Fields): dữ liệu mô tả trạng thái của đối tượng.
  • Phương thức (Methods): hành vi của đối tượng.
  • Constructor: khởi tạo đối tượng.
  • Static members: thành phần chia sẻ cho tất cả đối tượng.
class Circle { static double PI = 3.14; double r; Circle(double r){ this.r = r; } double area(){ return PI * r * r; } }

3. Access Modifiers (Phạm vi truy cập)

  • private: chỉ truy cập được trong cùng lớp.
  • protected: trong cùng package hoặc lớp con.
  • default: trong cùng package.
  • public: mọi nơi trong chương trình.

Quy tắc tốt: Thuộc tính nên là private, hành vi cần thiết mới để public.

4. Constructor và Destructor

Constructor là phương thức đặc biệt có cùng tên với lớp, dùng để khởi tạo đối tượng. Có thể overload nhiều constructor khác nhau.

class Point { int x, y; Point(){x=0;y=0;} Point(int x, int y){this.x=x;this.y=y;} }

Java không có destructor như C++; việc thu hồi bộ nhớ do Garbage Collector xử lý.

5. Từ khóa this

Từ khóa this tham chiếu đến chính đối tượng hiện tại – dùng khi tên biến trùng tham số hoặc khi gọi constructor khác.

class Account { String owner; Account(String owner){ this.owner = owner; } }

6. Overloading và Overriding

Overloading: cùng tên nhưng khác tham số trong cùng lớp.

Overriding: lớp con ghi đè phương thức lớp cha.

class MathUtils { int add(int a,int b){return a+b;} double add(double a,double b){return a+b;} }

7. Static members

Static thuộc về lớp, không thuộc về đối tượng.

class Counter { static int count=0; Counter(){count++;} static void showCount(){ System.out.println("Objects: "+count); } }

8. Ví dụ tổng hợp

class Employee { private static int nextId = 1; private int id; private String name; private double salary; Employee(String name, double salary){ this.id = nextId++; this.name = name; this.salary = salary; } void raise(double percent){ salary += salary*percent/100; } void show(){ System.out.println(id+" - "+name+": "+salary); } public static void main(String[] args){ Employee e1 = new Employee("An", 5000); Employee e2 = new Employee("Bình", 7000); e1.raise(10); e2.raise(5); e1.show(); e2.show(); System.out.println("Tổng nhân viên: "+nextId); } }

Kết luận

Kiến thức về lớp và đối tượng là trung tâm của Java. Việc thiết kế lớp đúng cách giúp mã dễ bảo trì, tái sử dụng và mở rộng.
